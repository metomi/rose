Rose User Guide: Getting Started

Introduction

If Rose is not installed at your site, please refer to Rose: Installation for detail.

Before using Rose, it is desirable to configure a few things for yourself. If you have just installed Rose at your site, you should configure it to meet your site requirements.

The format of both the user and site configuration file is documented in user/site config.

Rose combines the settings in the site configuration and the user configuration at run time. You can view the actual setting by issuing the command:

rose config

User Configuration

You can configure Rose by creating and editing your user configuration file at ~/.metomi/rose.conf. E.g.:

mkdir -p ~/.metomi
${EDITOR:-vi} ~/.metomi/rose.conf

Rose should work out of the box if it is configured correctly at your site.

However, you may want to change the [external] geditor (the external text editor used by GUIs) and the [external] editor (the external text editor used by CLI commands) settings. If these settings are not defined in the user and site configuration, the default external text editor used by GUIs is gedit. The default external text editor used by CLI commands is the value of the VISUAL or EDITOR environment variable, or vi if neither environment variable is set. If, for example, you want to use gvim as your text editor, you would add the following to your ~/.metomi/rose.conf:

[external]
editor=gvim -f
geditor=gvim -f

Site Configuration

You can configure Rose for your site by creating and editing the etc/rose.conf file in a Rose distribution.

Essential setup includes changing the options in [rose-host-select] and [rose-suite-run] to appropriate hostnames. You should also probably change the path settings in [rose-task-run] to their equivalent for your site.

If you are using Rosie, you will need to change the paths, urls, and hostnames in the rosie sections as well.

You may want to make sure the default programs referred to in [external] are sensible.

Plugins

You may want to set up cylc-specific text editor plugins - see the Cylc User Guide for details.

There are gedit, kate, and vim plugins for syntax highlighting of Rose configuration files, located at:

  • $ROSE_HOME/etc/rose-conf.lang
  • $ROSE_HOME/etc/rose-conf.xml
  • $ROSE_HOME/etc/rose-conf.vim

$ROSE_HOME should be replaced with the path to the root directory of your local Rose installation, which you can locate by running rose --version. See the files themselves for setup notes.
