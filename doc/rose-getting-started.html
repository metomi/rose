Rose Documentation

Rose User Guide: Getting Started

Introduction

If Rose is not installed at your site, please refer to Rose: Installation for detail.

Before using Rose, it is desirable to configure a few things for yourself. If you have just installed Rose at your site, you should configure it to meet your site requirements.

User Configuration

You can configure Rose by creating and editing a configuration file in your homespace (your user config):

~/.metomi/rose.conf

The format of this file is documented in user/site config.

You can view the current configuration options of Rose by opening a terminal and running:

rose config

You can override the options in these sections by entering your own options in your user config file. For example, changing the default graphical text editor can be done by putting the following lines in your user config file:

[external]
geditor=gvim

This will change the graphical text editor used by Rose to 'gvim'.

You can also override many internal constants of the rose config editor and rosie go. To change the keyboard shortcut of the Find Next action in the config editor to F3, put the following lines in your user config file:

[rose-config-edit]
accel-find-next=F3

This will be applied the next time you run the config editor.

Site Configuration

You can configure Rose for your site by editing the file in a Rose distribution:

etc/rose.conf

The format of this file is documented in user/site config.

Essential setup includes changing the options in [rose-host-select] and [rose-suite-run] to appropriate hostnames. You should also probably change the path settings in [rose-task-run] to their equivalent for your site.

If you are using Rosie, you will need to change the paths, urls, and hostnames in the rosie sections as well.

You may want to make sure the default programs referred to in [external] are sensible.
